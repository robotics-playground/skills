# Workspace and Build System Reference

> ROS 2 Jazzy | colcon, ament, package creation, dependencies

---

## 1. Workspace Layout

A ROS 2 workspace is a directory containing source code and build artifacts. The
standard layout is:

```
my_robot_ws/                   # Workspace root
  src/                         # Source packages (you write code here)
    package_a/
    package_b/
    package_c/
  build/                       # Build artifacts (auto-generated by colcon)
  install/                     # Installed packages (auto-generated by colcon)
  log/                         # Build logs (auto-generated by colcon)
```

**Only the `src/` directory is version-controlled.** The `build/`, `install/`, and
`log/` directories are generated by `colcon build` and should be in `.gitignore`.

```gitignore
# .gitignore for ROS 2 workspace
build/
install/
log/
```

## 2. Overlay / Underlay Concept

ROS 2 uses a layered sourcing model. Think of it like environment variable precedence -- higher layers override lower ones.

```
Layer 3 (highest priority): Your workspace     → source install/setup.bash
Layer 2:                     Team shared pkgs   → source /opt/team_ws/setup.bash
Layer 1 (lowest priority):   ROS 2 Jazzy base  → source /opt/ros/jazzy/setup.bash
```

**You MUST source in order, from bottom to top:**

```bash
# Always source the base ROS 2 installation first
source /opt/ros/jazzy/setup.bash

# Then source your workspace overlay
cd ~/my_robot_ws
colcon build
source install/setup.bash
```

**The #1 build gotcha:** After running `colcon build`, you MUST run
`source install/setup.bash` for changes to take effect. If you edit code and
rebuild but don't re-source, you're still running the old version.

**Pro tip:** Add to your `~/.bashrc`:
```bash
source /opt/ros/jazzy/setup.bash
# Optionally auto-source your workspace:
# source ~/my_robot_ws/install/setup.bash
```

## 3. Creating Packages

### Python Package (ament_python)

```bash
cd ~/my_robot_ws/src
ros2 pkg create --build-type ament_python --node-name my_node my_package \
  --dependencies rclpy std_msgs sensor_msgs
```

This creates:

```
my_package/
  my_package/                  # Python module (same name as package)
    __init__.py
    my_node.py                 # Your node
  resource/
    my_package                 # Marker file for ament
  test/
    test_copyright.py
    test_flake8.py
    test_pep257.py
  package.xml                  # Package metadata and dependencies
  setup.py                     # Python build configuration
  setup.cfg                    # Entry points and install config
```

### C++ Package (ament_cmake)

```bash
cd ~/my_robot_ws/src
ros2 pkg create --build-type ament_cmake --node-name my_node my_cpp_package \
  --dependencies rclcpp std_msgs sensor_msgs
```

This creates:

```
my_cpp_package/
  include/
    my_cpp_package/            # Public headers
  src/
    my_node.cpp
  CMakeLists.txt
  package.xml
```

### Message-Only Package (ament_cmake -- required for messages)

```bash
cd ~/my_robot_ws/src
ros2 pkg create --build-type ament_cmake my_robot_msgs \
  --dependencies std_msgs geometry_msgs sensor_msgs
```

Then add msg/srv/action directories and configure CMakeLists.txt.
See `references/communication.md` for message definition details.

### When to Use ament_python vs ament_cmake

| Choose `ament_python` | Choose `ament_cmake` |
|---|---|
| Pure Python nodes | C++ nodes |
| Quick prototyping | Performance-critical code |
| ML/AI integration | Custom message definitions |
| Team's primary language | Hardware drivers (if in C++) |
| No custom messages in this package | Mixed C++/Python in one package |

**Important:** Custom message packages MUST use `ament_cmake` even if your nodes
are in Python. Message generation requires CMake.

## 4. package.xml (Format 3)

The package manifest declares metadata and dependencies.

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd"
  schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_robot_perception</name>
  <version>0.1.0</version>
  <description>Perception nodes for the robot</description>
  <maintainer email="team@example.com">Robot Team</maintainer>
  <license>Apache-2.0</license>

  <!-- Build system -->
  <buildtool_depend>ament_python</buildtool_depend>

  <!-- Runtime dependencies (needed at build AND run time) -->
  <depend>rclpy</depend>
  <depend>sensor_msgs</depend>
  <depend>geometry_msgs</depend>
  <depend>cv_bridge</depend>
  <depend>my_robot_msgs</depend>

  <!-- Build-only dependencies -->
  <build_depend>some_build_tool</build_depend>

  <!-- Run-only dependencies -->
  <exec_depend>ros2launch</exec_depend>
  <exec_depend>rviz2</exec_depend>

  <!-- Test dependencies -->
  <test_depend>ament_copyright</test_depend>
  <test_depend>ament_flake8</test_depend>
  <test_depend>ament_pep257</test_depend>
  <test_depend>python3-pytest</test_depend>
  <test_depend>launch_testing</test_depend>

  <export>
    <build_type>ament_python</build_type>
  </export>
</package>
```

### Dependency Types

| Tag | When Package is Needed |
|---|---|
| `<depend>` | Build + Run time |
| `<build_depend>` | Build time only |
| `<exec_depend>` | Run time only |
| `<test_depend>` | Test time only |
| `<buildtool_depend>` | Build system itself |

**Most common mistake:** Forgetting to add a dependency to `package.xml`. If your
Python code imports from a package, it MUST be listed as `<depend>` or
`<exec_depend>`.

## 5. setup.py and setup.cfg (Python Packages)

### setup.py

```python
from setuptools import find_packages, setup

package_name = 'my_robot_perception'

setup(
    name=package_name,
    version='0.1.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        # Install marker file for ament
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        # Install package.xml
        ('share/' + package_name, ['package.xml']),
        # Install launch files
        ('share/' + package_name + '/launch',
            ['launch/perception.launch.py']),
        # Install config files
        ('share/' + package_name + '/config',
            ['config/camera_params.yaml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='Robot Team',
    maintainer_email='team@example.com',
    description='Perception nodes for the robot',
    license='Apache-2.0',
    tests_require=['pytest'],
    entry_points={
        'console_scripts': [
            # Format: 'executable_name = package.module:function'
            'camera_node = my_robot_perception.camera_node:main',
            'lidar_node = my_robot_perception.lidar_processor:main',
            'detector_node = my_robot_perception.detector:main',
        ],
    },
)
```

### setup.cfg

```ini
[develop]
script_dir=$base/lib/my_robot_perception

[install]
install_scripts=$base/lib/my_robot_perception
```

**The entry_points are critical.** Without them, `ros2 run` can't find your node.
Each entry maps a command name to a `module:function` path.

```bash
# This works because of the entry_points mapping:
ros2 run my_robot_perception camera_node
# Calls my_robot_perception.camera_node:main()
```

## 6. CMakeLists.txt Patterns (C++ Packages)

### Minimal C++ Package

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_robot_control)

# Default to C++17
if(NOT CMAKE_CXX_STANDARD)
  set(CMAKE_CXX_STANDARD 17)
endif()

# Find dependencies
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(geometry_msgs REQUIRED)
find_package(sensor_msgs REQUIRED)

# Build executable
add_executable(motor_controller src/motor_controller.cpp)
ament_target_dependencies(motor_controller
  rclcpp geometry_msgs sensor_msgs
)

# Install executable
install(TARGETS motor_controller
  DESTINATION lib/${PROJECT_NAME}
)

# Install launch files
install(DIRECTORY launch/
  DESTINATION share/${PROJECT_NAME}/launch
)

# Install config files
install(DIRECTORY config/
  DESTINATION share/${PROJECT_NAME}/config
)

# Testing
if(BUILD_TESTING)
  find_package(ament_lint_auto REQUIRED)
  ament_lint_auto_find_test_dependencies()
endif()

ament_package()
```

### Message Package CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_robot_msgs)

find_package(ament_cmake REQUIRED)
find_package(rosidl_default_generators REQUIRED)
find_package(std_msgs REQUIRED)
find_package(geometry_msgs REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/Detection.msg"
  "msg/Report.msg"
  "srv/AnalyzeArea.srv"
  "srv/GetStatus.srv"
  "action/InspectArea.action"
  DEPENDENCIES std_msgs geometry_msgs
)

ament_package()
```

## 7. colcon Build Commands

### Essential Commands

```bash
# Build everything in the workspace
cd ~/my_robot_ws
colcon build

# Build a single package (fastest for iterating)
colcon build --packages-select my_robot_perception

# Build with symlink install (Python changes take effect without rebuild)
colcon build --symlink-install

# Build with verbose output (for debugging build errors)
colcon build --event-handlers console_direct+

# Build in parallel with limited jobs (if RAM is tight)
colcon build --parallel-workers 2

# Build and run tests
colcon build && colcon test
colcon test-result --verbose  # Show test results
```

### Key Build Options

| Option | Effect | When to Use |
|---|---|---|
| `--packages-select pkg` | Build only the named package | Iterating on one package |
| `--packages-up-to pkg` | Build the package and all its dependencies | First build of a package with deps |
| `--symlink-install` | Symlink Python files instead of copying | Python development (no rebuild for code changes) |
| `--cmake-args -DCMAKE_BUILD_TYPE=Debug` | Debug build for C++ | Debugging C++ nodes with gdb |
| `--cmake-args -DCMAKE_BUILD_TYPE=Release` | Optimized C++ build | Performance testing |
| `--event-handlers console_direct+` | Show all build output | Debugging build errors |
| `--parallel-workers N` | Limit parallel jobs | Low-RAM systems (Raspberry Pi) |

### The --symlink-install Trick

For Python development, `--symlink-install` is essential. It creates symlinks to
your source files instead of copying them. This means:

```bash
# First build with symlink install
colcon build --symlink-install

# Now edit your Python code...
# No need to rebuild! Just re-run.
ros2 run my_package my_node

# But you DO need to rebuild if you:
# - Add new files or entry points
# - Change setup.py, setup.cfg, or package.xml
# - Change C++ code
# - Change message definitions
```

## 8. Common Build Errors and Fixes

| Error | Cause | Fix |
|---|---|---|
| `Could not find a package configuration file provided by "X"` | Missing dependency in package.xml or not installed | Add `<depend>X</depend>` to package.xml; `sudo apt install ros-jazzy-X` |
| `No module named 'X'` at runtime | Missing exec_depend or workspace not sourced | Add to package.xml; re-source install/setup.bash |
| `SetuptoolsDeprecationWarning` | Old setuptools | Ignore (harmless) or upgrade setuptools |
| `Multiple packages found with the same name` | Duplicate package in workspace | Remove duplicate from src/ |
| `Could not find a configuration file for package "X" that is compatible` | Version mismatch | Check if package is built for your ROS 2 distro |
| Entry point not found with `ros2 run` | Missing or incorrect entry_points in setup.py | Check console_scripts mapping; rebuild |
| `Package 'X' not found` in launch file | Package not built or not sourced | `colcon build --packages-select X && source install/setup.bash` |
| `CMake Error: The source directory does not appear to contain CMakeLists.txt` | Wrong build type in package.xml | Ensure `<build_type>` matches actual build system |
| `Import error` after adding new message | Message package not rebuilt or not sourced | Rebuild message package; re-source; rebuild dependent packages |
| Build succeeds but old code runs | Didn't re-source after build | `source install/setup.bash` |

## 9. Workspace Best Practices

### Do

- **One responsibility per package** -- a camera driver package, a perception package,
  a navigation package. Not one monolith.
- **Separate message packages** -- put custom messages in their own package
  (e.g., `my_robot_msgs`) so any package can depend on them without circular deps.
- **Use `--symlink-install`** for Python development to avoid rebuilding on every change.
- **Source the workspace** in every new terminal session.
- **Keep packages small** -- 3-5 nodes per package is a good target.
- **Use a bringup package** for launch files and configs that orchestrate the whole system.
- **Pin ROS 2 distro** -- everyone on the team uses the same Jazzy version.

### Don't

- **Don't mix ament_python and ament_cmake** in the same package (unless truly needed
  for pybind11 bindings). Use separate packages.
- **Don't put all nodes in one package** -- this forces a full rebuild when anything changes.
- **Don't import Python modules across packages** -- communicate through ROS 2 interfaces
  (topics, services, actions). If you need shared utilities, make a separate utility package.
- **Don't commit build/, install/, log/** -- add to .gitignore.
- **Don't use absolute paths** in launch files or configs -- use `get_package_share_directory()`.
- **Don't forget to rebuild message packages** when you change .msg/.srv/.action files --
  and rebuild all packages that depend on them.

## 10. Recommended Workspace Setup for Your Team

```bash
# Create the workspace
mkdir -p ~/my_robot_ws/src
cd ~/my_robot_ws/src

# Create packages
ros2 pkg create --build-type ament_cmake my_robot_msgs \
  --dependencies std_msgs geometry_msgs sensor_msgs

ros2 pkg create --build-type ament_python my_robot_bringup \
  --dependencies rclpy

ros2 pkg create --build-type ament_python my_robot_description

ros2 pkg create --build-type ament_python my_robot_perception \
  --dependencies rclpy sensor_msgs cv_bridge my_robot_msgs

ros2 pkg create --build-type ament_python my_robot_navigation \
  --dependencies rclpy geometry_msgs nav_msgs my_robot_msgs

ros2 pkg create --build-type ament_python my_robot_control \
  --dependencies rclpy geometry_msgs sensor_msgs

ros2 pkg create --build-type ament_python my_robot_inspection \
  --dependencies rclpy my_robot_msgs sensor_msgs

ros2 pkg create --build-type ament_python my_robot_teleop \
  --dependencies rclpy geometry_msgs

# Build everything
cd ~/my_robot_ws
colcon build --symlink-install

# Source the workspace
source install/setup.bash

# Verify
ros2 pkg list | grep my_robot
```
